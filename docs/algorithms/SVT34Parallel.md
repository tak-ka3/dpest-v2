# SVT34Parallel

## アルゴリズムの説明

SVT34Parallelは、SVT3またはSVT4の並列版です。複数のSVTインスタンスを並列に実行します。

**プライバシー保証**: プライバシー違反があります（理論 ε = ∞）。

## モード

**サンプリングモード**

## プライバシー損失結果

| 項目 | 値 |
|------|-----|
| 入力サイズ | 10 |
| 推定 ε | inf |
| 理論 ε | ∞ |
| 誤差 | 正確に検出 |
| 実行時間 | 105.22秒 |

**データソース**: `docs/privacy_loss_report.md`

**解釈**: DPESTは無限大のプライバシー損失を正確に検出。

## 理論的な計算量

**サンプリングモード**: $O(N \times m \times p)$ where $p$ は並列数

## 理論と実験結果の比較分析

### 比較: DP-Sniper vs StatDP vs DPEST

| 手法 | 推定 ε | 実行時間 |
|------|--------|----------|
| DP-Sniper | 0.268 | 180秒 |
| StatDP | 0.137 | 10800秒 |
| DPEST | inf | 105.22秒 |

**結論**: DPESTは無限大のプライバシー損失を正確に検出。他手法は有限値を推定し、プライバシー違反を見逃しています。実行時間も1.7-103倍高速。
